Hello Hexo

随口说说

今天开始正式写些东西，根据hexo和fluid的文档把基础的东西配了一下，感觉还挺好看的

安装

去官网安装nodejs和git这是必备的，安装完成后

npm i hexo -g 
hexo init blog ## 初始化blog

这是npm给的下载5.2.0版本的指令，不明白为啥npm install hexo -g 这条指令会下载4.2.0版本，明明啥区别的。

相关hexo指令

hexo init ##初始化
hexo new [layout] [tittle] ##新建页面/文章
-p --path ##指定路径
-r --replace ##替换同名
-s --slug ##没看懂这句是干啥的
hexo generate/g ##生成网页
-d --deploy ##部署
-w --watch ##观察文件改动
-f --force ##强制重新生成文件
hexo publish [layout] <filename> ##发布草稿
hexo server ##启动服务，本地加载
hexo deploy ##部署网站
hexo clean ##清空缓存文件

然后是fluid这是它的官方手册其中有很多详细的介绍，我只是进行了初步的改动，之后再打算详细的设置一下，比如统计啊，还有各个页面的壁纸高度不对（看着是真的别扭）

服务器设置

首先安装nginx，并配置开机自启

apt-get install nginx
apt install sysv-rc-conf
sysv-rc-conf nginx on

然后配置远程仓库

# 随便在哪里创建都可以
cd /blog
# 初始化一个仓库 
git init --bare .git 
cd /.git/hooks
#这个文件中的脚本会监听这个git的仓库的每一次push上来的事件，push到这个仓库一次，里面的脚本就执行一次

然后在新建一个post-receive，写入

#!/bin/sh
# 前面的/blog/hexo与nginx配置挂载的目录一致
git --work-tree=~/blog/hexo --git-dir=~/blog/hexo.git checkout -f

赋予权限

chmod 777 post-receive

nginx配置
在/etc/nginx/sites-enabled/defaulrt写入

server {
 listen  80; #再写一个443
 root   /blog/hexo; # 这里的意思是挂载/blog/hexo目录

 location / {
      index  index.html index.htm;
 }
}

配置本地hexo的_config.yml

deploy:
type: git 
repo: 用户名@ip地址:/www/hexo.git
branch: master

然后尝试推送就好了
