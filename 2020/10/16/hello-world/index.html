Hello Hexo

随口说说

今天开始正式写些东西，根据hexo和fluid的文档把基础的东西配了一下，感觉还挺好看的

安装

去官网安装nodejs和git这是必备的，安装完成后

npm i hexo -g 
hexo init blog ## 初始化blog

这是npm给的下载5.2.0版本的指令，不明白为啥npm install hexo -g 这条指令会下载4.2.0版本，明明啥区别的。

相关hexo指令

hexo init ##初始化
hexo new [layout] [tittle] ##新建页面/文章
-p --path ##指定路径
-r --replace ##替换同名
-s --slug ##没看懂这句是干啥的
hexo generate/g ##生成网页
-d --deploy ##部署
-w --watch ##观察文件改动
-f --force ##强制重新生成文件
hexo publish [layout] <filename> ##发布草稿
hexo server ##启动服务，本地加载
hexo deploy ##部署网站
hexo clean ##清空缓存文件

然后是fluid这是它的官方手册其中有很多详细的介绍，我只是进行了初步的改动，之后再打算详细的设置一下，比如统计啊，还有各个页面的壁纸高度不对（看着是真的别扭）还有在设置好之前，不要使用开那个强制https，会取不到资源

服务器设置

首先安装nginx，最好采用编译安装

wget http://nginx.org/download/nginx-1.18.0.tar.gz
tar xzvf nginx-1.15.2.tar.gz
cd nginx-1.15.2
./configure // 如果后面还想要配置 SSL 协议，就执行后面一句！
./configure --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_ssl_module --with-file-aio --with-http_realip_module
make && make install

配置远程仓库

useradd git
passwd git // 设置密码
su git // 这步很重要，不切换用户后面会很麻烦
cd /home/git/
mkdir blog // 项目存在的真实目录
mkdir repos && cd repos
git init --bare blog.git // 创建一个裸露的仓库
cd blog.git/hooks
vi post-receive // 创建 hook 钩子函数，输入了内容如下

然后在post-receive写入

#!/bin/sh
git --work-tree=/home/git/blog --git-dir=/home/git/repos/hexo.git checkout -f

赋予权限

chmod +x post-receive
exit // 退出到 root 登录
chown -R git:git /home/git/repos/blog.git // 添加权限

可以测试下仓库可以使用了没有

git clone git@server_ip:/home/git/repos/blog.git

安全起见关闭git的shell登录权限

cat /etc/shells // 查看 git-shell 是否在登录方式里面
which git-shell // 查看是否安装
vi /etc/shells
添加上2步显示出来的路劲，通常在 /usr/bin/git-shell

修改/etc/passwd中的权限

// 将原来的
git:x:1000:1000::/home/git:/bin/bash

// 修改为
git:x:1000:1000:,,,:/home/git:/usr/bin/git-shell

调整nginx的配置文件

nginx -s stop // 先停止nginx
cd /usr/local/nginx/conf
vi nginx.conf
修改 root 解析路径 /home/git/blog
同时将 user 改为 root 不然nginx无法访问 
nginx -s reload

_config.yml 推送设置

deploy:
type: git 
repo: git@server_ip:~/repos/blog.git
branch: master

github page/gitee page

为什么我会写继续写这章，原因是我的域名是在阿里云购买的也做了备案，但是服务器是百度云的，最近的新政策，如果要接入百度云的服务器，必须在百度云再备案一次，最后想了想还是不在服务器上挂博客了，所以打算将域名绑在github page服务和gitee page服务上，分别对应着境外和国内的访问。

对于github创建一个username.github.io的仓库就可以使用pege服务了，并跟据下图配置完成一定要点save，这时候访问username.github.io就可以访问自己的页面，个鬼啊

这个同样得设置_config.yml，然后推送一下

对于gitee，创建一个与用户名相同的仓库，或者从github上直接fork过来，然后改成用户名

在这里开启page服务，然后就可以通过username.gitee.io访问自己的网页了(推送啊！推送)
